2.vscode_main

Posted on 2025-11-19

1. 目的

我习惯使用 pycharm 的快捷键。

比如 我编写 python 的时候，输入 main
pycharm 会自动补全为 if __name__ == \"__main__\":
我觉得这个功能，非常基础，但是 vscode 中没找到类似的。

我的目的:

  1. 在 vscode 中实现，我输入 main 后，可以自动补全 为:
    if __name__ == \"__main__\":

2. 方案

2.1 找到 configure Snippets

快捷键 [ctrl] + [shift] + [P] 调出 preference 界面。

查找 configure snippets.

2.2 指定 python 的 snippets

输入 python 设置 python 的 snippets(代码片段)

2.3 编写 snippets

加入 main 相关指定代码

2.4 测试结果

输入 main ，提示 有 snippets

然后按 [tab] 可以自动补全

1.vscode-rsync 远程连接(服务器)

Posted on 2025-11-19

1. 目的

我使用过 vscode 和 pycharm 的远程编写代码/远程调试的功能。

vscode: 通过安装 remote 插件，在远程端(服务器) 执行一个监听的脚本。
特点:

  1. 没有本地代码，所有代码都在远程端(服务器)，所有代码都是及时同步的。
  2. CPU 占用极高，如果远程端(服务器)，只有 2 核 + 2G，很快达到 CPU 占用 100%，出现无法重启的问题。
  3. 当代码量巨大的时候，打开文件会出现问题。

PyCharm: 通过 rsync 插件(专业版自带)，在远程端(服务器)进行执行 rsync 指令。
特点:

  1. 有本地代码，代码需要本地编写，通过指令 rsync 上传到远程端(服务器)。
    [专业版能在文件保存的同时自动进行上传]。
  2. 但是并不能及时同步。rsync 的同步是单向的。默认是 [本地端]->[远程端] 同步。
    比如，你的代码生成了一张图片，这张图片只存在[远程端]。
    解决方案是: 手动指定同步方向 [远程端]->[本地端]。

我非常喜欢 pycharm 远程编写代码的模式，但是我也很喜欢 vscode 占用内存小，大量的开源工具。

我的目的是:

  1. 在 vscode 中实现 pycharm 远程代码同步的功能。

2. 解决方案

vscode 的插件库除了 remote-ssh 其实还有 rsync。
但是 rsync 的下载量不大。
我使用的是 vscode-ext.sync-rsync

2.1 setting.json

主要的内容是编写 setting.json 文件。

红色部分为 rsync 相关的 参数。
在绿色部分中填写自己的远程 IP 地址。

2.2 免密 ssh

建议设置免密的 ssh 登录。

  1. 在本地生成密钥[本地端执行]
    ssh-keygen -t rsa -b 4096
  2. 拷贝公钥到远端[本地端执行]
    ssh-copy-id root@远程IP
